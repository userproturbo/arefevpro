ТЕХНИЧЕСКОЕ ЗАДАНИЕ

на разработку фронтенда сайта в стиле roman jeanelie**

1. Общее описание проекта

У заказчика есть рабочий бэкенд с авторизацией, лайками, комментариями и загрузкой контента через админку.
Задача — полностью заменить текущий минимальный дизайн на новый фронтенд, выполненный в стилистике сайта:
👉 https://www.romanjeanelie.com должно быть почти 1:1

При этом все текущие возможности бэкенда должны быть сохранены, без изменений логики.

Фронтенд создаётся на базе следующей структуры разделов:

Главная

Мои проекты

Фото

Видео

Музыка

Блог

Каждый раздел имеет собственный контент и должен корректно загружаться с API.

2. ДИЗАЙН (UI/UX)
2.1. Общий стиль

Фронтенд должен повторять эстетику сайта romanjeanelie.com:

Бруталистичная типографика

Жёсткий контраст (чёрный / белый / красный)

Огромные шрифты

Много пространства

Минималистичные интерфейсные элементы

Вертикальное меню слева

Акцент на анимации переходов (шторка, появление контента)

Большие визуальные блоки (фото, превью постов)

WebGL/Canvas/градиентная фоновая сцена (по желанию, можно упростить, если нужно)

Стиль должен быть современным, динамичным, но не перегруженным.

2.2. Главное меню

Меню слева в вертикальной ориентации, как у ориентира.

Пункты:

HOME

Мои проекты

Фото

Видео

Музыка

Блог

Меню всегда фиксировано слева.
Активный пункт подсвечивается (цвет/полоса).

2.3. Главная страница

На сайте-примере внизу расположен большой заголовок "CREATIVE DEVELOPER ROMAN JEAN-ELIE".

У нас:

👉 Вместо текста должна быть лента из фотографий, движущаяся / перелистывающаяся / появляющаяся с анимацией.
Лента — горизонтальная или в несколько рядов (решает дизайнер).

Главная должна производить эффект портфолио-галереи, а не текстового интро.

2.4. Анимация при переходе в раздел (“чёрная шторка”)

При клике на пункт меню появляется:

✔ анимированная чёрная шторка, перекрывающая экран
✔ на шторке отображается список постов (или альбомов)

Поведение:

Шторка появляется с GSAP/Framer Motion анимацией (Curtain Reveal)

Слева — список постов/альбомов

Справа — превью контента

Это ключевой элемент дизайна → нужно повторить максимально близко к оригиналу.

2.5. Разделы
Мои проекты / Фото / Видео / Музыка / Блог

В зависимости от типа контента:

А) Блог

На шторке — список заголовков постов
Справа — превью постов:

обложка

начало текста / вступление

дата

Б) Фото

На шторке — список альбомов
Справа — превью альбомов (сетка или крупные карточки)

При входе в альбом — большая галерея, лайки, комментарии.

В) Видео

На шторке — список видео-проектов
Справа — видеопревью

Г) Музыка

Шторка — список аудио-треков/релизов
Справа — карточки с визуализацией/обложкой

Д) Мои проекты

То же, что на сайте-примере — огромный список названий проектов слева + превью справа.

3. ФРОНТЕНД-ФУНКЦИОНАЛ
3.1. При клике на пункт меню:

Запуск анимации шторки

Загрузка списка постов/альбомов через API

Отображение контента справа

Важно:
переходы должны быть плавными, быстрыми, кинематографичными.

3.2. Страница поста / фото / видео

Каждый элемент должен иметь:

лайки

комментарии

возможность залогиниться

форма добавления комментария

счётчик лайков

Функционал — полностью сохраняется с текущего сайта.

3.3. Авторизация

Пользователь может:

Регистрация (если есть)

Логин

Оставлять комментарии

Ставить лайки

Пользователи НЕ могут загружать контент.

4. ИНТЕГРАЦИЯ С БЭКЭНДОМ
Требования к фронтенд-программисту:

Использовать существующий API (форматы ответов менять нельзя)

Все эндпоинты должны быть согласованы с заказчиком

Вся логика лайков/комментариев остаётся прежней

Админка остаётся нетронутой

Фронтенд только отображает данные
